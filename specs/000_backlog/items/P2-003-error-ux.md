# P2-003: API エラー時のユーザーフレンドリーな表示

## 概要

現在、API エラー（タイムアウト、レート制限、無効な API キーなど）はコンソールまたは簡易的なエラーメッセージで表示される。ユーザーに分かりやすいエラーメッセージと再試行ボタンを提供する。

## 受け入れ条件

1. OpenCode Go API エラー時に、エラーの種類に応じたメッセージが表示される
   - レート制限: 「リクエストが多すぎます。しばらく待ってから再試行してください。」
   - タイムアウト: 「応答に時間がかかりすぎました。再試行してください。」
   - 無効な API キー: 「API キーが無効です。管理者にお問い合わせください。」
2. エラーメッセージに「再試行」ボタンが付く
3. エラーからの回復後、正常にチャットを継続できる
