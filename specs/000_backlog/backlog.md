## P1 — High（次のスプリントで実装）

### P1-001: 画像入力（マルチモーダル）フロントエンド UI

**背景:**
バックエンド (`chat.ts`) は既に `imageBase64` に対応しているが、フロントエンドに画像アップロード UI が存在しない。ドラッグ＆ドロップまたはファイルピッカーで画像を選択し、Base64 エンコードして送信できるようにする。

**受け入れ条件:**
1. チャット入力欄に画像アップロードボタン（クリップアイコン）が表示される
2. 画像ファイルを選択するとプレビューが表示される
3. 画像付きメッセージを送信すると、API リクエストに `imageBase64` が含まれる
4. 5MB を超える画像はアップロード前に拒否される

**関連:** FR-006, [Phase 2 仕様](../002-chat-app-phase2/spec.md#1-マルチモーダル非対応モデル警告)

**関連ファイル:**
- `frontend/src/components/Chat/ChatInput.tsx`
- `frontend/src/hooks/useChat.ts`
- `frontend/src/services/chatApi.ts`

---

### P1-002: マルチモーダル非対応モデル警告（FR-014）

**背景:**
OpenCode Go API でマルチモーダル対応なのは `glm-5.2` と `glm-5.1` のみ。それ以外のモデル（Kimi, DeepSeek, Qwen, MiniMax, MiMo, Hy3）で画像付きメッセージを送信すると、画像が無視される可能性がある。

**受け入れ条件:**
1. 非マルチモーダルモデル選択中に画像をアップロードすると、警告バナーが表示される
2. マルチモーダルモデル選択中に画像をアップロードすると、警告は表示されない
3. 警告には「画像を削除」「このまま送信」の選択肢がある

**関連:** [Phase 2 仕様 §1](../002-chat-app-phase2/spec.md#1-マルチモーダル非対応モデル警告)

**関連ファイル:**
- `frontend/src/components/Chat/ChatInput.tsx`
- `frontend/src/App.tsx`

---

### P1-003: ストリーミング中断時の中間保存

**背景:**
現在の `useChat.ts` の `stop()` は AbortController で SSE を切断するのみ。生成途中のテキストは DB に保存されないため、ページリロード後に失われる。

**受け入れ条件:**
1. 停止ボタンクリック時、生成途中のテキストが DB に保存される
2. ページリロード後、中断時点までの AI 応答が表示される
3. まだ 1 文字も受信していない状態で停止した場合、「(生成が中断されました)」と保存される

**関連:** [Phase 2 仕様 §2](../002-chat-app-phase2/spec.md#2-ストリーミング中断時の中間保存)

**関連ファイル:**
- `frontend/src/hooks/useChat.ts`
- `backend/src/routes/chat.ts`（新規 `POST /api/chat/stop`）

---

### P1-004: 会話タイトルの自動生成

**背景:**
現在、新規会話のタイトルは固定の「New Chat」のまま。ユーザーが手動でタイトルを編集する機能もない。最初のユーザーメッセージからタイトルを自動生成するか、AI に要約させることで、会話一覧の識別性を向上させる。

**受け入れ条件:**
1. 新規会話作成後、最初のメッセージ送信時にタイトルが自動生成される
2. 生成されたタイトルは会話一覧に反映される
3. タイトル生成に失敗しても、チャット機能は継続して動作する

**実装案:**
- 最初のユーザーメッセージを 20 文字で切ってタイトルにする（簡易実装）
- または、AI に「以下のメッセージから 20 文字以内のタイトルを生成してください」と問い合わせる

**関連ファイル:**
- `frontend/src/hooks/useConversations.ts`
- `backend/src/routes/conversations.ts`

## P2 — Medium（計画的に実装）

### P2-001: 長い会話履歴の警告

**背景:**
各モデルには `contextLength` 制限がある。会話が長くなると古いメッセージが切り捨てられ、意図しない応答が返る可能性がある。推定トークン数がコンテキスト上限の 80% を超えた場合に警告を表示する。

**受け入れ条件:**
1. 推定トークン数がコンテキスト上限の 80% を超えると、画面上部に警告バナーが表示される
2. 80% 未満の場合、警告は表示されない
3. 警告バナーは閉じることができる（そのセッション中のみ非表示）

**関連:** [Phase 2 仕様 §3](../002-chat-app-phase2/spec.md#3-長い会話履歴の警告)

**関連ファイル:**
- `backend/src/routes/conversations.ts`
- `frontend/src/components/Chat/ChatMessageList.tsx`

---

### P2-002: 複数タブ間の状態同期

**背景:**
同一ブラウザで複数タブを開いている場合、タブ A で新規会話を作成してもタブ B のサイドバーに反映されない。`BroadcastChannel` を使用してタブ間の状態を同期する。

**受け入れ条件:**
1. タブ A で会話を作成すると、タブ B のサイドバーに自動的に追加される
2. タブ A で会話を削除すると、タブ B からも削除される
3. タブ B でアクティブにしていた会話がタブ A で削除された場合、タブ B は空のチャット画面を表示する
4. `BroadcastChannel` 非対応ブラウザではエラーなく動作する（手動リロードが必要なのみ）

**関連:** [Phase 2 仕様 §4](../002-chat-app-phase2/spec.md#4-複数タブ間の状態同期)

**関連ファイル:**
- `frontend/src/hooks/useBroadcastSync.ts`（新規）
- `frontend/src/hooks/useConversations.ts`
- `frontend/src/App.tsx`

---

### P2-003: API エラー時のユーザーフレンドリーな表示

**背景:**
現在、API エラー（タイムアウト、レート制限、無効な API キーなど）はコンソールまたは簡易的なエラーメッセージで表示される。ユーザーに分かりやすいエラーメッセージと再試行ボタンを提供する。

**受け入れ条件:**
1. OpenCode Go API エラー時に、エラーの種類に応じたメッセージが表示される
   - レート制限: 「リクエストが多すぎます。しばらく待ってから再試行してください。」
   - タイムアウト: 「応答に時間がかかりすぎました。再試行してください。」
   - 無効な API キー: 「API キーが無効です。管理者にお問い合わせください。」
2. エラーメッセージに「再試行」ボタンが付く
3. エラーからの回復後、正常にチャットを継続できる

**関連ファイル:**
- `frontend/src/hooks/useChat.ts`
- `frontend/src/components/Chat/ChatMessageList.tsx`

---

### P2-004: 画像サイズ制限のフロントエンドバリデーション

**背景:**
仕様上は 5MB 以下の画像のみアップロード可能としているが、現在フロントエンドでサイズチェックが行われていない（画像アップロード自体未実装だが、P1-001 実装時に合わせて対応が必要）。

**受け入れ条件:**
1. 5MB を超える画像を選択すると、エラーメッセージが表示されアップロードされない
2. 5MB 以下の画像は正常にアップロードできる

**関連:** P1-001

**関連ファイル:**
- `frontend/src/components/Chat/ChatInput.tsx`

### P2-005: 会話の手動リネーム

**背景:**
自動生成タイトル（P1-004）に加え、ユーザーが会話一覧から直接タイトルを編集できるようにする。

**受け入れ条件:**
1. サイドバーの会話タイトルをクリックすると編集モードに入る
2. Enter で保存、Esc でキャンセル
3. 空のタイトルは保存されない（前のタイトルを維持）

**関連ファイル:**

### P3-006: 長い会話の仮想スクロール

**背景:**
会話が数百メッセージに及ぶと、DOM 要素が増大しレンダリング性能が低下する。`react-window` または `react-virtuoso` を使用して仮想スクロールを実装する。

**受け入れ条件:**
1. 100 件以上のメッセージがあっても、スクロールが 60fps で滑らかである
2. 新規メッセージ追加時、自動スクロールが正しく動作する
3. コードブロックのシンタックスハイライトが仮想スクロール後も正しく表示される

**関連ファイル:**
- `frontend/src/components/Chat/ChatMessageList.tsx`

## P4 — Nice to have（アイデア段階）

### P4-001: PWA（Progressive Web App）対応

**背景:**
`vite-plugin-pwa` を使用して、オフラインキャッシュやホーム画面へのインストールを可能にする。

**受け入れ条件:**
1. Lighthouse の PWA チェックで全項目合格
2. ホーム画面に追加できる
3. オフライン時に過去の会話を閲覧できる（Service Worker キャッシュ）

---

### P4-002: プロンプトテンプレート

**背景:**
よく使うプロンプト（翻訳、要約、コードレビューなど）をテンプレートとして保存・再利用できる。

**受け入れ条件:**
1. テンプレート一覧を管理できる（追加・編集・削除）
2. チャット入力欄からテンプレートを選択して挿入できる
3. テンプレートは `localStorage` に保存される

---

### P4-003: コードブロックの実行機能

**背景:**
AI が生成した JavaScript/Python コードを、ブラウザ上または Sandpack で実行できるようにする。

**受け入れ条件:**
1. コードブロックに「実行」ボタンが表示される（対応言語のみ）
2. 実行結果がコードブロック下に表示される
3. 実行はサンドボックス化され、ブラウザに影響を与えない

---

### P4-004: 音声入力（Web Speech API）

**背景:**
ブラウザの `SpeechRecognition` API を使用して、音声でメッセージを入力できるようにする。

**受け入れ条件:**
1. マイクボタンをクリックして音声入力を開始できる
2. 音声認識結果がテキスト入力欄に反映される
3. 認識中は「聞き取り中…」のインジケーターを表示する

---

### P4-005: 会話の共有（リンク生成）

**背景:**
特定の会話を読み取り専用の公開リンクとして共有できるようにする。

**受け入れ条件:**
1. 会話メニューから「共有リンクを作成」を選択できる
2. リンクを開くと、ログイン不要で会話内容が閲覧できる
3. 共有リンクは有効期限（30日）を持つ
